# UI UX Rules

- Read context/ui-tokens.md before UI work.
- Read context/ui-registry.md before building components.
- Keep the interface professional and calm.
- Use high contrast.
- Use large touch targets.
- Avoid decorative UI.
- Priority 1 and 2 must be visually prominent.
- Offline state must feel safe.
- Validation must be inline.
- Do not use technical error messages.
- Prefer fewer screens and fewer steps.
